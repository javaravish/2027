1. Generics

Generics in Java allow you to write code that works with different data types without specifying the exact type in advance, making your code more flexible, reusable, and type-safe. Generics were introduced in Java 5 to address shortcomings of pre-generic Java in handling objects, specifically regarding type safety and reusability.

Why use generics?

Generics offer several key advantages:

- Type Safety: Generics enable the compiler to perform type checking at compile time, catching errors that would otherwise manifest at runtime as ClassCastException .

- Code Reusability: You can write a single class, interface, or method that operates on various data types, reducing code duplication.

- No Explicit Type Casting: Generics eliminate the need for manual type casting when retrieving objects, resulting in cleaner and more readable code.

- Improved Code Readability and Maintainability: By specifying types explicitly, generic code becomes easier to understand and maintain.

- Generic Algorithms: Generics enable the creation of algorithms that work across various types, promoting efficient coding practices.

How generics work

Generics work by using type parameters—placeholders for actual data types. These type parameters are defined within angle brackets < >. For examples of generic classes and methods, you can refer to the provided source documents.

Key concepts

- Type Erasure: During compilation, the Java compiler replaces generic type parameters with their bounds (or Object if no bounds are specified), ensuring backward compatibility but limiting runtime type information.

- Bounded Type Parameters: You can restrict the types that can be used as type arguments using the extends keyword (upper bound) or super keyword (lower bound).

- List<? extends Number>: Accepts Number or any of its subclasses (e.g., Integer, Double).

- List<? super Integer>: Accepts Integer or any of its superclasses (e.g., Number, Object).

- Wildcards: Represented by the ? symbol, wildcards can specify unknown types in situations like method parameters or field declarations, increasing flexibility.

- List<?>: Represents a list of unknown type.

Limitations

- Generics cannot directly work with primitive types (e.g., int, char, double); you must use their corresponding wrapper classes (e.e., Integer, Character, Double).

- You cannot create instances of a generic type parameter (e.g., new T()).

- Due to type erasure, you cannot use instanceof with parameterized types at runtime.

Generics are a vital part of modern Java development, significantly improving the safety, reusability, and readability of your code. Understanding their principles and mastering their usage is crucial for any Java programmer.
