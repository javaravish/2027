1. Introduction to OOP in Java

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects" which can contain data (attributes) and code (methods). Java is fundamentally object-oriented, with all code written inside classes.

2. Core OOP Concepts in Java

1. Class
A class is a blueprint or template for creating objects. It defines:

• The data/attributes (fields/variables) that each object will have

• The behavior/actions (methods/functions) that each object can perform

2. Object
An object is an instance of a class. It's a concrete entity created from the class blueprint, with its own set of values for the attributes.

Example:

        
// Class definition
class Car {
    // Fields (attributes)
    String color;
    String model;
    int year;

    // Method
    void startEngine() {
        System.out.println("Engine started for " + model);
    }
}

public class Main {
    public static void main(String[] args) {
        // Creating an object
        Car myCar = new Car();
        myCar.color = "Red";
        myCar.model = "Tesla Model S";
        myCar.year = 2023;

        myCar.startEngine(); // Output: Engine started for Tesla Model S
    }
}

3. Abstraction

Hides complex implementation details and shows only essential features. Reduces complexity and increases efficiency.

is a technique by which we can hide the unnecessary data that is not needed from a user and expose only that data is of interest to the user. It hides all unwanted data so that users can work only with the required data.

Implementation Methods:

1. Abstract classes: Can have both abstract and concrete methods

2. Interfaces: Pure abstraction (until Java 7)

Why Use Abstraction?

• Reduce complexity: Hide unnecessary details

• Security: Expose only what's needed

• Flexibility: Change implementation without affecting users

4. Encapsulation

Bundling data (attributes) and methods (behaviors) that operate on that data into a single unit (class), while restricting direct access to some components.

Protects data by restricting direct access and provides controlled access through methods. Prevents unauthorized access and modification.

Through encapsulation, data is hidden and protected from access by outside non-member methods of a class. Only member methods defined in a class will have access to the data.

Why Use Encapsulation?

• Data protection: Prevent unauthorized access

• Controlled access: Validate data before modification

• Flexibility: Change implementation without affecting other code

Example:

        
class BankAccount {
    private double balance; // private field

    // Public method to access private field
    public double getBalance() {
        return balance;
    }

    // Public method to modify private field with validation
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        }
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount();
        account.deposit(1000);
        // account.balance = -500; // Error: balance is private
        account.withdraw(200);
        System.out.println("Balance: " + account.getBalance());
    }
}

5. Polymorphism

The ability of objects to take on many forms (same method name behaves differently based on the object).

Types of Polymorphism:

1. Compile-time (Method Overloading): Multiple methods with same name but different parameters

2. Runtime (Method Overriding): Child class provides specific implementation of parent's method

Why Use Polymorphism?

• Flexibility: Write code that works with objects of different classes

• Extensibility: Easily add new classes without modifying existing code

6. Inheritance

A mechanism where a new class (child/subclass) inherits properties and behaviors from an existing class (parent/superclass).

Promotes code reuse and establishes relationships between classes. Child classes inherit fields and methods from parent classes.

Inheritance in Java is a process where a child class acquires all the properties and behaviors of the parent class. The existing class is called parent class and the new class is called child class.

Why Use Inheritance?

1. Code Reusability: Avoid duplicating code

2. Method Overriding: Enable runtime polymorphism

3. Class Hierarchy: Create logical relationships

4. Extensibility: Easily add new features to existing code

Types of Inheritance in Java

1. Single Inheritance

2. Multilevel Inheritance

3. Hierarchical Inheritance

4. Multiple Inheritance (Through Interfaces)

Java doesn't support multiple inheritance with classes but allows it with interfaces

Java intentionally does not support multiple inheritance of classes (where a class can inherit from multiple parent classes) to avoid complexity and ambiguity in code.

The Diamond Problem: Core Issue

The primary reason is the "diamond problem", a classic multiple inheritance ambiguity that occurs when:

If:

- Class A has a method print()

- Classes B and C override print()

- Class D inherits from both B and C

Ambiguity: Which print() should D inherit - B's version or C's version?

3. Association, Aggregation, and Composition

1. Association

• Bi-directional relationship between objects

• Objects have their own lifecycle

• Example: Teacher and Student

        
class Teacher {
    void teach(Student s) {
        System.out.println("Teaching student");
    }
}

class Student {
    void learn(Teacher t) {
        System.out.println("Learning from teacher");
    }
}

2. Aggregation (Weak "has-a")

• One object "has" another object

• Child can exist without parent

• Example: Department has Professors

        
class Professor {
    String name;
    Professor(String name) { this.name = name; }
}

class Department {
    List<Professor> professors;
    Department(List<Professor> profs) {
        professors = profs;
    }
}

3. Composition (Strong "has-a")

• One object "owns" another object

• Child cannot exist without parent

• Example: Car has an Engine

        
class Engine {
    void start() { System.out.println("Engine started"); }
}

class Car {
    private Engine engine;
    Car() {
        engine = new Engine(); // Engine created with Car
    }
    void startCar() {
        engine.start();
    }
}
