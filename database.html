1. What is a Database?

A database is an organized collection of data that can be easily accessed, managed, and updated.

Databases are used to store information like customer records, employee data, etc.

Example: A table storing employee records (ID, name, salary).

2. What is DBMS and RDBMS?

DBMS (Database Management System) is software that interacts with the user, applications, and the database to capture and analyze data.

Examples: MySQL, Oracle, SQL Server, PostgreSQL.

RDBMS (Relational Database Management System) is a type of DBMS that stores data in tables (rows and columns) and uses SQL for database access.

Feature DBMS RDBMS
Full Form Database Management System Relational Database Management System
Structure File-based Table-based (rows & columns)
Relation No relation between tables Tables have defined relationships
Example Microsoft Access MySQL, PostgreSQL, Oracle

SQL (Structured Query Language) is used to interact with databases ‚Äî for querying, updating, inserting, and deleting data.

SQL (Structured Query Language) is a standard language for managing and manipulating relational databases. It's used for querying, inserting, updating, and deleting data.

  
    SELECT * FROM employees;

5. What is a Primary Key?

A primary key is a column or set of columns that uniquely identifies each row in a table. It cannot contain NULL values and must be unique.

  
    CREATE TABLE students (
        student_id INT PRIMARY KEY,
        name VARCHAR(50)
    );

6. What is a Foreign Key?

A foreign key is a column or set of columns that establishes a relationship between data in two tables. It references the primary key of another table.

  
    CREATE TABLE orders (
        order_id INT,
        customer_id INT,
        FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
    );

7. What is Normalization?

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. The main normal forms are:

Form Rule Example
1NF Atomic values only
2NF No partial dependency on composite key
3NF No transitive dependency
BCNF (Boyce-Codd Normal Form) BCNF (Boyce-Codd Normal Form)

8. What is Denormalization?

Denormalization is the process of combining normalized tables to improve read performance at the cost of some redundancy.

9. What is an Index?

An index is a database object that improves the speed of data retrieval operations on a table.

It works like a book's index, allowing the database to find data without scanning the entire table.

  
    CREATE INDEX idx_name ON employees(name);

10. What is a View?

A view is a virtual table based on the result set of an SQL statement. It doesn't store data itself but presents data from underlying tables.

  
    CREATE VIEW high_salary AS
    SELECT name, salary FROM employees WHERE salary > 50000;

11. What is a Stored Procedure?

A stored procedure is a group of SQL statements stored in the database and executed as a unit.

A stored procedure is a prepared SQL code that you can save and reuse. It can accept parameters and return values, improving performance and security.

  
    CREATE PROCEDURE GetAllEmployees()
    BEGIN
        SELECT * FROM employees;
    END;

12. What is a Trigger?

A trigger is a special kind of stored procedure that automatically executes when an event (INSERT, UPDATE, DELETE) occurs on a table.

  
    CREATE TRIGGER update_salary
    BEFORE UPDATE ON employees
    FOR EACH ROW
    SET NEW.update_at = NOW();

13. What is the difference between WHERE and HAVING ?

  • WHERE filters rows before grouping (GROUP BY)
  • HAVING filters groups after the GROUP BY clause
  • WHERE cannot use aggregate functions, while HAVING can
Clause Used With Filters Based On
WHERE SELECT, UPDATE Rows before grouping
HAVING GROUP BY Groups after aggregation 
  
    SELECT dept, COUNT(*)
    FROM employees
    GROUP BY dept
    HAVING COUNT(*) > 5;

14. What is a Join? What are its types?

Joins combine rows from two or more tables based on a related column.

  • INNER JOIN: Returns matching rows from both tables
  • LEFT JOIN: Returns all rows from left table and matched rows from right
  • RIGHT JOIN: Returns all rows from right table and matched rows from left
  • FULL JOIN: Returns all rows when there's a match in either table
  • CROSS JOIN: Returns Cartesian product of both tables
  
    SELECT e.name, d.name
    FROM employees e
    JOIN departments d ON e.dept_id = d.id;

15. What is the difference between DELETE and TRUNCATE and DROP?

  • DELETE: Removes rows from a table (can be rolled back, can have WHERE clause)
  • TRUNCATE: Removes all rows from a table (faster than DELETE, cannot be rolled back)
  • DROP: Removes the entire table structure from the database
Operation DELETE TRUNCATE
Deletes Row-by-row All rows at once
Rollback Yes No
Speed Slower Faster

16. What is ACID in DBMS?

A transaction is a sequence of operations performed as a single logical unit of work. It follows ACID properties:

Letter Stands For Meaning
A Atomicity All operations complete or none
C Consistency Database remains valid after transaction
I Isolation Transactions occur independently
D Durability Changes persist even after system failure

17. What is a Transaction?

A transaction is a sequence of one or more SQL operations performed as a single unit of work.

  
    BEGIN;
    UPDATE accounts SET balance = balance - 100 WHERE id = 1;
    UPDATE accounts SET balance = balance + 100 WHERE id = 2;
    COMMIT;

19. What is the difference between UNION and UNION ALL?

  • UNION combines results of two queries and removes duplicates
  • UNION ALL combines results including duplicates (faster than UNION)

20. What are aggregate functions in SQL?

Aggregate functions perform calculations on sets of values:

  • COUNT(): Counts rows
  • SUM(): Calculators sum
  • AVG(): Calculators average
  • MAX(): Finds maximum value
  • MIN(): Finds minimum value

5. SQL JOINS

JOINS are used to combine rows from two or more tables based on a related column between them.

Sample Tables Used

employees Table

id name dept_id
1 Ravi 10
2 Anu 20
3 John NULL
4 Priya 30

departments Table

dept_id dept_name
10 HR
20 IT
30 Finance
40 Marketing

1. INNER JOIN

Returns only the matching rows from both tables.

  
    SELECT e.name, d.dept_name
    FROM employees e
    INNER JOIN departments d ON e.dept_id = d.dept_id;

‚úî Output:

name dept_name
Ravi HR
Anu IT
Priya Finance

Note: John (no department) and Marketing (no employee) are excluded.

2. LEFT JOIN (LEFT OUTER JOIN)

Returns all rows from the left table ( employees ), and matching rows from the right table. If no match, NULL is returned.

  
    SELECT e.name, d.dept_name
    FROM employees e
    LEFT JOIN departments d ON e.dept_id = d.dept_id;

‚úî Output:

name dept_name
Ravi HR
Anu IT
John NULL
Priya Finance

Note: John has no department, so dept_name is NULL .

3. RIGHT JOIN (RIGHT OUTER JOIN)

Returns all rows from the right table ( departments ), and matching rows from the left table. If no match, NULL is returned.

  
    SELECT e.name, d.dept_name
    FROM employees e
    RIGHT JOIN departments d ON e.dept_id = d.dept_id;

üìå Output:

name dept_name
Ravi HR
Anu IT
Priya Finance
NULL Marketing

üìå Note: No employee is in Marketing department, so name is NULL.

4. FULL JOIN (FULL OUTER JOIN)

Returns all rows from both tables. Non-matching rows will have NULL values.

Not supported in MySQL directly (can simulate using UNION).

  
    -- Simulated FULL JOIN using UNION in MySQL
    SELECT e.name, d.dept_name
    FROM employees e
    LEFT JOIN departments d ON e.dept_id = d.dept_id
    UNION
    SELECT e.name, d.dept_name
    FROM employees e
    RIGHT JOIN departments d ON e.dept_id = d.dept_id;

üìå Output:

name dept_name
Ravi HR
Anu IT
John NULL
Priya Finance
NULL Marketing

5. CROSS JOIN

Returns the Cartesian product ‚Äì every row from the first table is paired with every row from the second.

  
    SELECT e.name, d.dept_name
    FROM employees e
    CROSS JOIN departments d;

üìå Output (4 employees x 4 departments = 16 rows):

name dept_name
Ravi HR
Ravi IT
Ravi Finance
Ravi Marketing
Anu HR
Anu IT
... ...

6. SELF JOIN

Used to join a table to itself.

Example: Find employees who have the same department.

  
    SELECT e1.name AS Emp1, e2.name AS Emp2, e1.dept_id
    FROM employees e1
    JOIN employees e2 ON e1.dept_id = e2.dept_id
    WHERE e1.id <> e2.id;
